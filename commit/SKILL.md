# 提交修改的内容到git

- 检查当前项目git仓库中发生变更的文件
- 检查是否有需要添加到.gitignore的文件
- 提交前将变更摘要写入 CLAUDE.md 文件
- 只提交，不要push

## 提交流程

### 1. 检查变更
- 执行 `git status` 查看变更的文件
- 执行 `git diff` 查看具体的变更内容

### 2. 生成变更摘要
- 从 `git diff` 中提取本次修改的主要内容
- 生成简洁的变更摘要（1-2句话）

### 3. 更新 CLAUDE.md
- CLAUDE.md 文件位置：
  - 项目根目录(针对小项目)
  - 子项目根目录(针对大型项目包含很多独立组建的情况)
- 根据当前git变更的文件属于哪个组建, 更新对应组建根目录中的CLAUDE.md文件
- 如果文件不存在，则创建它
- 在 `## 变更记录` 章节下添加新行，格式：
  ```
  YYYY-MM-DD HH:MM:SS - 变更摘要
  ```
- 如果 `## 变更记录` 章节不存在，则创建
- 如果 `## 变更记录` 中已经写入了本次变更的摘要, 则直接使用该摘要, 不用重写

### 4. 提交代码
- 执行 `git add` 和 `git commit`
- 提交信息使用变更摘要

### 5. 确认状态
- 执行 `git status` 确认提交成功
